¿Qué es n8n? En lenguaje humano.

💡 La Gran Analogía

Imagina que eres el director de una fábrica. La fábrica tiene una cinta transportadora (el workflow) y a lo largo de ella hay estaciones de trabajo especializadas (los nodos). Cada estación recibe algo, lo transforma o lo envía a otro lugar, y lo pasa a la siguiente. Tú no operas ninguna máquina: tú diseñas cómo se conectan y le dices a Claudio que las construya y las configure.

🗒 El Workflow (El Mapa)

Es el plano completo de tu automatización. Define el orden en que las tareas se ejecutan, desde el primer disparador hasta la acción final. Piénsalo como la receta completa de un platillo: ingredientes, pasos, y resultado final. Puedes tener muchos workflows distintos para distintos procesos de tu empresa.

🔌 El Nodo (La Estación)

Es una tarea individual y especializada. Hay nodos para todo: recibir correos, consultar bases de datos, llamar a una API, transformar datos, esperar un tiempo, enviar mensajes de WhatsApp, etc. Cada nodo hace una sola cosa y muy bien. Se conectan entre sí para formar el workflow.

El Diccionario del Director

Estos son los términos que Claudio te pedirá con frecuencia. No necesitas ser técnico para entenderlos; solo necesitas saber qué buscar en tu cuenta o plataforma.

Activación

⚡ Trigger (Disparador)

Es el evento que arranca tu workflow. Sin un trigger, el flujo nunca se ejecuta. Es la primera pieza en la cadena.

Analogía: Es como el timbre de tu puerta: alguien toca (el evento) y tú abres y actúas (el workflow se ejecuta).

Ejemplos de triggers comunes:

  • Webhook: Alguien llama a una URL especial → el workflow arranca
  • Schedule / Cron: Se ejecuta automáticamente cada X tiempo (cada hora, cada lunes a las 8am)
  • Email Trigger: Llega un correo → arranca el workflow
  • Form Trigger: Alguien llena un formulario → arranca
Credencial

🔑 API Key (Llave de Acceso)

Una contraseña especial que le das a Claudio (y a n8n) para que pueda hablar con servicios externos en tu nombre. No es tu contraseña normal de usuario; es una clave técnica que genera cada plataforma.

Analogía: Es como una tarjeta magnética de empleado. No es tu contraseña de casa; es un acceso específico para que n8n entre a, por ejemplo, tu cuenta de Gmail o tu base de datos.

¿Dónde la encuentras? Generalmente en la configuración de cada servicio, bajo secciones llamadas: "API Keys", "Developer Settings", "Tokens", o "Integraciones".

Configuración

🌐 URL de Instancia

La dirección web de tu n8n. Es como la dirección postal de tu fábrica. Puede ser una URL pública si usas n8n en la nube (ej. https://tu-empresa.app.n8n.cloud) o una dirección local si lo tienes en tu computadora (http://localhost:5678).

¿Para qué la necesitas? Para que Claudio sepa exactamente dónde crear, activar o modificar tus workflows.
Datos

📋 JSON (El formato de los datos)

Es el idioma en que los nodos se hablan entre sí. Cuando un nodo le pasa información al siguiente, lo hace en formato JSON. No necesitas leerlo ni escribirlo, pero debes saber que existe y que cuando Claudio habla de "campos" o "propiedades" se refiere a los datos dentro de un JSON.

Analogía: Es como una etiqueta de paquetería con campos: Nombre, Dirección, Peso, Destino. Cada nodo lee la etiqueta, hace su trabajo, y puede añadir o modificar campos antes de pasar el paquete al siguiente.
Lógica

🔳 IF / Switch (Toma de decisiones)

Nodos especiales que permiten que tu workflow tome caminos distintos dependiendo de una condición. Como decir "si el cliente es VIP, manda correo especial; si no, manda el estándar".

Analogía: Es el guardia de seguridad del aeropuerto: revisa tu boleto y te manda al carril de primera clase o al estándar según lo que dice.
Programación

🕒 Cron Expression (Programación de tiempo)

Una forma técnica de decirle a n8n cuándo ejecutar un workflow automáticamente. Tú no necesitas escribirla; solo necesitas describir cuándo quieres que corra y Claudio la genera.

Ejemplo de cómo pedirlo: "Claudio, quiero que esto se ejecute todos los lunes a las 9am" — Claudio escribe la expresión cron correspondiente.
Integración

👥 Webhook (Receptor de llamadas)

Una URL especial y única que n8n te genera. Cuando alguien (o algún sistema) la "llama" enviando datos, tu workflow se activa con esos datos. Es el trigger más poderoso y versátil.

Analogía: Es como una línea directa de teléfono exclusiva para tu workflow. Cualquiera que marque ese número (con la información correcta) activa la línea de producción.
Expresión

📝 Expression (Valor dinámico)

Una mini-instrucción que le dice a un nodo: "no uses un valor fijo, usa el valor que viene del nodo anterior". Se escribe entre llaves dobles: {{ ... }}

Ejemplo: En vez de poner "Estimado Juan" en un correo, pones {{ $json.nombre }} y el correo automáticamente usa el nombre de cada persona de tu lista. Claudio siempre sabe cómo escribirlas.
Arquitectura

🔒 Credential (Credencial guardada)

En n8n, las credenciales (API Keys, contraseñas de bases de datos, tokens OAuth) se guardan de forma segura y separada del workflow. Un nodo referencia una credencial por nombre, pero no guarda la clave en sí. Esto es más seguro y reutilizable.

¿Qué debes hacer? Crear la credencial en n8n con los datos de acceso antes de que Claudio construya el workflow, y decirle el nombre que le pusiste.
Proceso

🔁 Loop / Iterate (Repetición)

Cuando tu workflow necesita hacer lo mismo con muchos elementos (ej. enviar un correo a cada cliente de una lista de 500), un Loop le dice a n8n que procese cada elemento uno a uno o en grupos.

Analogía: Es como una cadena de montaje que produce 500 autos en serie. Cada auto pasa por los mismos pasos, pero son unidades independientes.

Cómo hablar con Claudio

La clave para obtener el mejor resultado es dar órdenes que incluyan qué quieres hacer, con qué herramientas (nodos), y cuál es la condición o contexto. A continuación, plantillas de instrucción probadas.

🆕 Crear un flujo nuevo

Tu instrucción Plantilla base
Claudio, crea un workflow en n8n que haga lo siguiente:

1. Se active cada lunes a las 8am (trigger de schedule)
2. Lea los datos de mi hoja de Google Sheets llamada "Clientes activos"
3. Para cada cliente, genere un resumen personalizado usando OpenAI GPT-4
4. Envíe ese resumen por Gmail al correo de cada cliente

Mis credenciales en n8n están guardadas como: "Google Sheets - Principal" y "OpenAI - Empresa".
💡 Nota: Siempre menciona el nombre exacto de tus credenciales guardadas en n8n para que Claudio las referencie correctamente.

🚫 Diagnosticar un error ("entra por arriba y sale por abajo")

💬 ¿Qué significa "entra por arriba y sale por abajo"?

En n8n, cuando un nodo tiene un error, los datos "salen" por la conexión de error (roja, abajo) en vez de continuar el flujo normal (verde, arriba). El workflow falla y se detiene en ese nodo. Así es como describírselo a Claudio.

Tu instrucción Reporte de error
Claudio, mi workflow "Resumen semanal de clientes" está fallando. El nodo de Google Sheets muestra error y los datos salen por la conexión roja.

El mensaje de error que veo es: "[aquí pegas el mensaje de error que ves en pantalla]"

¿Puedes revisar la configuración de ese nodo y decirme qué está mal?
💡 Tip: Siempre copia y pega el mensaje de error exacto que aparece en rojo. Es la pista más valiosa para Claudio.

🔁 Cambiar un nodo por otro

Tu instrucción Sustitución de nodo
Claudio, en mi workflow "Notificaciones de ventas" actualmente estoy usando el nodo de Email (genérico/SMTP) para enviar correos.

Quiero reemplazarlo por el nodo de Gmail para poder usar mi cuenta de Google directamente.

Mi credencial de Gmail en n8n se llama "Gmail - Corporativo". Por favor, mantén el mismo asunto y cuerpo del correo que ya tenía el nodo anterior.
💡 Tip: Di siempre "mantén el contenido anterior" para que Claudio no pierda la lógica del correo al hacer el cambio.

🔳 Agregar una condición (bifurcación IF)

Tu instrucción Lógica de decisión
Claudio, después del nodo Google Sheets en mi workflow, necesito agregar una bifurcación:

— Si el campo "plan" del cliente es igual a "Premium", que siga a un nodo de Slack para notificar al equipo de ventas.
— Si es cualquier otro valor, que continúe al nodo de Gmail para enviar el correo estándar.

▶ Activar, pausar o ejecutar manualmente

Tu instrucción Control del workflow
Claudio, por favor activa el workflow llamado "Reporte diario de KPIs" en mi instancia de n8n. /

Claudio, desactívalo temporalmente, voy a modificar las credenciales.

Claudio, ejecútalo una vez manualmente ahora para que podamos probar si funciona con datos reales.

🔎 Entender qué hace un workflow existente

Tu instrucción Auditoría de workflow
Claudio, revisa el workflow que acabo de cargar y explícame en lenguaje simple:
— ¿Qué hace este workflow paso a paso?
— ¿Qué nodos usa y para qué sirve cada uno?
— ¿Qué credenciales necesita configuradas?
— ¿Hay algo que podría fallar o mejorar?

🌟 La fórmula de oro para dar órdenes

Trigger ¿Qué inicia el workflow? (tiempo, formulario, correo, llamada HTTP)
Fuente de datos ¿De dónde vienen los datos? (Google Sheets, base de datos, API, correo)
Transformación ¿Qué le pasa a los datos en el camino? (filtrar, resumir con IA, calcular)
Destino final ¿Dónde termina? (correo, Slack, base de datos, WhatsApp, reporte)
Credenciales Nombres exactos tal como los guardaste en n8n (ej. "Gmail - Corporativo")

Checklist de Supervisión

Antes de ejecutar cualquier workflow que Claudio construya, haz estas preguntas. Son las mismas que haría un ingeniero senior. No necesitas entender el código, solo confirmar los hechos.

📖 Antes de ejecutar: Revisar el diseño

⚡ ¿El Trigger es correcto? Pregunta: "Claudio, ¿qué inicia este workflow? ¿Cuándo exactamente se activa?" Confirma que coincide con lo que necesitas.
🔑 ¿Las credenciales son las correctas? Pregunta: "¿Qué credenciales usa este workflow?" Verifica que los nombres coincidan con los que tienes guardados en n8n.
📋 ¿Los datos fluyen en el orden correcto? Pregunta: "Explícame paso a paso qué información recibe y envía cada nodo." Si el orden no tiene sentido para tu proceso, corrígelo antes de ejecutar.
🚫 ¿Hay manejo de errores? Pregunta: "¿Qué pasa si el nodo de Google Sheets falla? ¿El workflow lo reporta o simplemente se detiene silenciosamente?"
🌐 ¿El destino final es el correcto? Pregunta: "¿A qué correo/canal/base de datos está enviando los resultados?" Confirma que no va a un ambiente de prueba o a la cuenta equivocada.

▶ Durante la prueba: Ejecución controlada

📊 Prueba con datos reales pero seguros Pide a Claudio que ejecute el workflow manualmente con un solo registro antes de procesar cientos. "Claudio, ejecuta esto con solo el primer registro de la hoja."
👀 Verifica el resultado en el destino Después de la prueba, ve al lugar donde debería haber llegado el resultado (tu correo, Slack, hoja de cálculo) y confirma que está correcto.
🕐 Revisa los logs de ejecución En n8n puedes ver la ejecución del workflow y si algún nodo falló. Pregunta a Claudio: "¿Todos los nodos terminaron en verde (sin error)?"

🔒 Preguntas de seguridad que siempre debes hacer

¿Este workflow modifica o borra datos? Si la respuesta es sí, pide siempre una prueba en modo "solo lectura" primero o en un ambiente de prueba.
¿Cuántos registros procesará en cada ejecución? Un workflow programado que procesa 10,000 registros por día puede tener costos de API inesperados. Confirma la escala.
¿Envía correos o mensajes en producción? Si el workflow envía notificaciones, confirma explícitamente que durante la prueba NO envíe mensajes reales a tus clientes.

🤔 Los 5 errores más comunes y cómo reportarlos

1. "No se encuentra la credencial"

Cómo reportarlo: "Claudio, el nodo X dice que no encuentra la credencial Y. ¿Cómo la creo en n8n?"

2. "Authentication failed" o "401 Unauthorized"

Cómo reportarlo: "Claudio, la API Key que configuré no funciona, el error dice 401." — Generalmente la clave expiró o es incorrecta.

3. "Cannot read property of undefined"

Cómo reportarlo: "Claudio, este error aparece en el nodo de Transform/Code. ¿Qué campo está buscando que no existe?" — Un campo de datos tiene nombre diferente.

4. El workflow se activa pero no hace nada

Cómo reportarlo: "Claudio, el workflow se ejecutó (veo la ejecución en el historial) pero no llegó ningún correo/mensaje. ¿Qué nodo falla en silencio?"

5. El workflow corre dos veces o en loop

Cómo reportarlo: "Claudio, el workflow se está ejecutando repetidamente o más veces de las esperadas." — Revisar el trigger y las condiciones de activación.
