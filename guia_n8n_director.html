Anatomía de n8n explicada al Director

Aprende de forma intuitiva cómo "piensa" n8n para que puedas dictarle arquitecturas exactas y robustas a Claudio.

1. El ADN de n8n: Items (JSON) 📦

El concepto más importante de n8n es este: TODOS los nodos se comunican pasando cajas de información llamadas "Items", las cuales están escritas siempre en formato JSON (pares de Nombre y Valor).

¿Qué es un Item / JSON?

Es simplemente un formato estandarizado para enviar datos. Si un formulario te envía una respuesta, n8n no la ve como texto, la ve como una pequeña cajita así:

{
  "nombre": "Juan Pérez",
  "empresa": "Acme Corp",
  "presupuesto": 5000
}

La Regla de Oro de los Nodos

Si a un nodo le llegan 5 Items (por ejemplo, le entran 5 correos), el nodo realizará su tarea 5 veces automáticamente.
Para decirle a Claudio: "Claudio, asegúrate de que el nodo de correo agrupe la información para enviar 1 solo correo, en lugar de ejecutar 1 correo por cada item que le llegó de la base de datos."

2. Los Nodos Core (Los engranes clave) ⚙️

Aunque como Director yo gestiono más de 1,000 aplicaciones, el 80% de un buen workflow se rige usando unos pocos nodos súper poderosos. Conoce sus roles:

Webhook / Triggers

Webhooks (Las Puertas de Entrada)

Un Webhook es literalmente una URL secreta. Funciona como un buzón que siempre está abierto escuchando si otra app "le avienta" datos en tiempo real.

Comunícalo: "Claudio, inicia el flujo usando un nodo Webhook POST para recibir la información de pagos desde nuestro servidor."
IF / Switch

Switch (El Semáforo Semántico)

Te permite enrutar datos a diferentes caminos según condiciones. Por ejemplo: Si el cliente es "VIP" que vaya al equipo A, si es "Estándar" al equipo B.

Comunícalo: "Mete un Switch Node que analice el 'país' del cliente. Enruta por diferentes canales si es México, España o Colombia."
Edit Fields (Set)

Edit Fields (El Traductor y Limpiador)

Antes se llamaba "Set Node". Es la herramienta para limpiar datos, renombrar variables o crear variables nuevas antes de mandarlas a otro software.

Comunícalo: "Usa un nodo de Edit Fields para formatear la fecha a dd/mm/aaaa antes de insertarla en Google Sheets."
HTTP Request

HTTP Request (El Comodín)

Si la aplicación que quieres usar no tiene un nodo nativo de n8n, usamos este nodo para comunicarnos con cualquier API del mundo.

Comunícalo: "La app X no tiene integración nativa. Revísale la documentación a su API y ármate un nodo HTTP Request con su Bearer Token."

3. Las Expresiones (Haciéndolo Dinámico) 🧬

Las Expresiones son la magia de n8n. Permiten que los campos se llenen solos con la información que pasó por un nodo anterior, en lugar de escribir textos fijos.

En el código interno de n8n, se ven así: {{ $json.nombre }}. Yo me encargaré de escribirlas por ti, simplemente debes saber que existen.

Ejemplos de cómo decírselo a Claudio:

  • "Claudio, en el campo de Asunto del correo, usa una expresión dinámica para que sea 'Hola [Nombre del usuario] tu pedido es el #[ID de pedido]' usando los datos del nodo anterior."
  • "Asegúrate de referenciar el email extrayéndolo correctamente del nodo de Stripe."

4. El Botón Nuclear: El Nodo de Código (Code Node) 👨‍💻

Gracias a mi base de conocimientos N8N Skills, sé utilizar perfectamente Python y JavaScript dentro de n8n.

Usamos el nodo de Código cuando la limpieza de datos es demasiado compleja (ej. agrupar matemáticamente finanzas, extraer palabras clave de un texto muy largo). Como Director, no hace falta que escribas el código, solo define la regla de negocio.

Comunícalo: "Claudio, los datos que salen del API están muy feos, son listas anidadas gigantes. Haz un nodo de Código (prefiero JavaScript) que itere esos items, extraiga solo el total general, lo sume, y me regrese un item limpio y único con el valor."

5. Limites y Reglas: Lo que SÍ y lo que NO 🚫

Es vital que como Director conozcas las fronteras de esta herramienta para evitar diseñar arquitecturas inviables o pedirle a Claudio cosas para las que n8n no fue hecho.

✅ Lo que n8n hace EXCELENTE

  • Orquestación (Backend): Conectar, por ejemplo, los pagos en Stripe con tu CRM y tu sistema de facturación.
  • Sincronización Masiva: Consumir miles de filas de un Excel o Base de Datos, transformarlas y cargarlas en otro sistema.
  • ETL (Extraer, Transformar y Cargar): Recibir datos en un idioma/formato complejo y convertirlos limpiamente a otro ecosistema.
  • Triggering por Eventos: Ejecutar secuencias exactas tipo "Si recibo un Webhook de un cliente nuevo, entonces actívate".

❌ Lo que n8n NO ES (Restricciones)

  • No es un Frontend (Web): No puedes crear aplicaciones con diseño web, botones e interfaces gráficas para tus clientes usando n8n. Es 100% "tras bambalinas".
  • No es una Base de Datos: n8n procesa información "en tránsito", pero no debe guardar grandes históricos de clientes en su base interna. Eso debe vivir en Airtable o SQL.
  • Evita Esperas Largas: Un workflow de n8n no debe quedarse literalmente "pausado esperando" 10 días a que alguien apruebe algo; corremos riesgo de Timeout. Es mejor romper eso en 2 workflows separados por un evento.
  • No es para Streaming/Gaming: Aunque es rápido, es un orquestador, por lo que no funciona como servidor de WebSockets en tiempo real de máxima exigencia para apps dinámicas.
